# YouTube UI Tests

Лабораторная работа по UI-тестированию.

## Стек
- Java 17
- Selenium WebDriver
- Maven

## Сценарий 1

Запуск без headless с авторизованным профилем Chrome:
1. Открыть YouTube
2. Выполнить поиск по запросу
3. Открыть первое видео
4. Проверить состояние лайка
5. Если лайк уже стоял — снять, обновить страницу и поставить заново
6. Обновить страницу и проверить, что лайк стоит

Для запуска используется заранее подготовленный профиль Chrome с выполненным входом.

### Запуск Chrome для сценария 1
Используется bat-файл:`open-youtube-debug-chrome.bat`

Он закрывает уже запущенные процессы Chrome и запускает браузер с включённым remote debugging на порту 9222. Также bat-файл использует отдельную папку профиля браузера, чтобы Selenium мог подключиться к уже авторизованной сессии для выполнения сценария 1.

После запуска Chrome запускается тест `YouTubeLikeTest` командой `.\mvnw.cmd -Dtest=YouTubeLikeTest test`

## Сценарий 2
Headless-сценарий для CI:
1. Открыть YouTube
2. Выполнить поиск по запросу
3. Открыть первое видео
4. Сохранить артефакты:
    - скриншот страницы
    - HTML страницы

### Локальный запуск

Запускается командой `.\mvnw.cmd -Dtest=YouTubeSmokeTest test -Dheadless=true.`

### CI

В репозитории настроен GitHub Actions workflow с ручным запуском (workflow_dispatch).

После выполнения workflow сохраняются артефакты:

`last-page.png`

`last-page.html`
